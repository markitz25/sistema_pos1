# Buenas prácticas de uso y seguridad

## Gestión de usuarios
- Crear un usuario distinto para cada persona.
- No compartir contraseñas ni guardarlas en texto plano.
- Asignar rol de administrador solo a quien lo necesite.
- Desactivar cuentas de trabajadores que ya no usan el sistema.

## Gestión de datos
- Mantener productos organizados y evitar duplicados.
- Revisar periódicamente el stock de productos clave.
- Actualizar datos de contacto de clientes cuando cambien.
- Registrar categorías de forma ordenada para facilitar reportes.

## Respaldo de información
- Programar respaldos de la base de datos (diarios o semanales según el volumen).
- Guardar respaldos en un lugar seguro y distinto al servidor principal.
- Probar restauraciones en un entorno de pruebas de vez en cuando.

## Seguridad básica
- Usar contraseñas seguras (mezcla de letras, números y símbolos) y cambiarlas con regularidad.
- No exponer el sistema directamente a internet sin medidas adicionales (firewall, VPN, etc.).
- Seguir las políticas de seguridad de la organización.

## Documentación y cambios
- Registrar cambios importantes del sistema (actualizaciones, nuevas funciones, configuración).
- Actualizar esta documentación y la base de conocimientos cuando se agreguen funcionalidades.
